# esBG-Reporting Project Setup

## Server Configuration

### Laravel Backend - Nginx Configuration
- **Port**: 89 (NOT 8000)
- **Server name**: esbg-reporting.localhost
- **Root directory**: /var/www/esBG-reporting/server/public
- **PHP handler**: PHP-FPM socket at /run/php/php8.4-fpm.sock
- **Config location**: /etc/nginx/sites-available/esbg-reporting.conf

```nginx
server {
    listen 89;
    server_name esbg-reporting.localhost;

    root /var/www/esBG-reporting/server/public;
    index index.php;

    # Logs
    access_log /var/log/nginx/esbg-reporting.access.log;
    error_log /var/log/nginx/esbg-reporting.error.log;

    # Gzip compression
    gzip on;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    # Static files - cache them
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    # Block access to sensitive files
    location ~ /\.(htaccess|env|git) {
        deny all;
    }

    # Route all requests through Laravel
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP handler - pass to PHP-FPM socket
    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.4-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        
        # Additional headers for Sanctum
        fastcgi_param HTTP_ACCEPT_LANGUAGE $http_accept_language;
    }
}
```

### Frontend - Nuxt Configuration
- **Port**: 3000 (development)
- **Base URL**: `http://esbg-reporting.localhost:89`
- **Environment variables**:
  - `NUXT_PUBLIC_API_BASE_URL`: Set to `http://esbg-reporting.localhost:89`

## Getting Started

### Prerequisites
- PHP 8.4 with FPM
- Nginx
- MariaDB/MySQL
- Node.js & npm
- Redis (optional, for caching)

### Setup Steps

1. **Backend (Laravel)**
   ```bash
   cd /var/www/esBG-reporting/server
   composer install
   php artisan migrate --seed
   php artisan cache:clear
   ```

2. **Frontend (Nuxt)**
   ```bash
   cd /var/www/esBG-reporting/client-frontend
   npm install
   npm run dev
   ```

3. **Services**
   ```bash
   # Start Nginx
   sudo systemctl start nginx
   
   # Start PHP-FPM
   sudo systemctl start php8.4-fpm
   
   # Start MariaDB
   sudo systemctl start mariadb
   ```

4. **Access the application**
   - Frontend: `http://localhost:3000`
   - API: `http://esbg-reporting.localhost:89`

## Important Notes
1. Laravel is accessed through Nginx on port 89, NOT port 8000
2. Nuxt frontend runs on port 3000 and communicates with API at `http://esbg-reporting.localhost:89`
3. Ensure the `/etc/hosts` file includes: `127.0.0.1 esbg-reporting.localhost`
4. Ensure Nginx and PHP-FPM are running
5. Make sure storage directories have correct permissions for www-data user

## Common Issues & Fixes

### 502 Bad Gateway
- **Cause**: PHP-FPM not running or Nginx socket configuration is incorrect
- **Fix**: 
  ```bash
  sudo systemctl restart php8.4-fpm
  sudo systemctl restart nginx
  ```

### Laravel temp directory errors
- **Cause**: Storage and bootstrap/cache directories owned by root, not www-data
- **Fix**: `chown -R www-data:www-data /var/www/esBG-reporting/server/storage /var/www/esBG-reporting/server/bootstrap/cache`
